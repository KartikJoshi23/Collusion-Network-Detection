CollusionGraph — Cross-Domain Collusion Detection Platform

One imbalance-robust, explainable, budget-evaluated graph stack that screens illicit-finance networks (crypto transaction graphs, synthetic AML worlds) and procurement bid-rigging cartels (EU award data, multi-country bid registers, OCDS feeds) through a single intermediate representation. Batch ML produces versioned artifacts; a torch-free read-only API serves them to an investigator console and an evidence-grounded LLM copilot. Screening signal only — the system never determines guilt.

Python 3.11 · PyTorch Geometric FastAPI · DuckDB · Parquet React 19 · Vite · Sigma.js · GSAP Docker · 3-service compose NVIDIA NIM LLM · guarded agent 373 backend tests · CI leakage gates

Data Flow

where data enters, what transforms it, what an analyst finally sees
1

Acquisition — scripts/download_data.py

Datasets download from their official sources (Mendeley API, Google Drive, Kaggle token, OpenTender), are sha-256 verified against committed manifests, and land in data/raw/. Licenses are recorded; raw data never enters git.

in: public sourcesout: data/raw/* + manifest verdicts
2

Ingestion — collusiongraph ingest

Six thin adapters map each dataset faithfully onto the shared IR (typed nodes/edges/labels/history). Domain quirks stay in raw_attrs; degradation paths (missing bidder identities, null buyers, undated edges) are explicit and tested.

in: data/rawout: data/interim/<ds>/*.parquet
3

Split & featurize — leakage firewall

Strict-inductive temporal splits (train graphs contain no test-period adjacency) or entity-disjoint LOCO/LOMO folds. Features are computed as-of the training cutoff; train-time normalization statistics are frozen and shipped with every checkpoint. A 12-test leakage suite with negative controls runs at split construction and in CI.

in: IR storeout: split reports · feature packs
4

Train & score — collusiongraph train

Config-shape dispatch routes one YAML to the right runner: baselines, GNNs, unsupervised arms, calibrated ensemble, injection recovery, transfer matrices, multi-seed / label-noise / label-efficiency campaigns. Every run writes run.json + test-window scores; campaigns are resumable.

in: configs/experiment/*.yamlout: eval_outputs/<run>/scores + run.json
5

Queue & explain — collusiongraph score / explain

Calibrated node scores roll up through Leiden communities into ranked alert queues (size-capped, NMS-deduplicated, both tx-level and actor-level granularities). Top alerts receive evidence bundles: PGExplainer subgraphs with fidelity checks, matched motifs, and FATF/OECD red-flag citations.

in: scores + IRout: alerts.parquet · explanations/*.json
6

Evaluate & publish — collusiongraph eval · poe demo-artifacts · poe paper-tables

The §4.5 harness is the only component allowed to compute published metrics. The artifact builder assembles serving.json (queues, bundles, metrics, rigor artifacts — exists-checked); the paper-tables generator renders LaTeX/markdown tables straight from the same artifacts, with a drift guard against the reproducibility map.

in: eval_outputs/*out: serving.json · paper tables
Trust boundary. Everything above this line is offline batch — it can run on a laptop, Colab, or a spot GPU and simply deposits files. Everything below is the always-on product surface: read-only, stateless, torch-free, and unable to write, retrain, or see raw features. Node raw-feature vectors, credentials, and training data never cross this boundary.
7

Serve — collusiongraph serve (FastAPI)

Seven read-only endpoint families over the artifact index. Subgraph responses are windowed server-side (bounded hops, node caps) so the browser never receives a full graph; alert ids are path-traversal-hardened; every JSON response carries the screening caveat.

in: serving.json + artifactsout: REST JSON
8

Interrogate — Investigator Copilot

Questions stream over SSE through a bounded agent loop: SELECT-only SQL over a DuckDB view of the queue, alert/bundle/metrics tools, and BM25 retrieval over the curated red-flag corpus. Deterministic guards rewrite guilt language and reject un-grounded numbers before anything is released; the trace and evidence render live in the dock.

in: analyst question + context alertout: grounded answer + evidence + trace
9

Triage — the analyst

Analysts work a fixed alert budget: ranked queue → windowed subgraph → evidence dossier → measured queue precision in the Model Lab, with multi-seed uncertainty, transfer matrices, and significance tests rendered beside the headline numbers. Output is an escalation decision by a human — the system's verdict is never more than a screening signal.

in: console + copilotout: human escalation decision

AWS Cloud Architecture

product deployment — batch/serving split maps 1:1 onto cheap cloud primitives

The architecture's core economic property: inference is batch/offline by design, so the always-on footprint is a static site, a tiny stateless API, and object storage — no GPU is ever provisioned 24/7. The same three Docker images that run docker compose up locally run unchanged in the cloud (dev/prod parity). Two tracks share one topology: a serverless launch tier that scales to zero, and a containerized growth tier for sustained traffic.

Why the numbers stay this small: the serving plane never touches a GPU — the single largest cost lever in ML products. Training/scoring is a scheduled batch job on ephemeral spot capacity, and everything user-facing is either static (CloudFront), scale-to-zero (Lambda), or a quarter-vCPU container. The three-image compose stack means the growth tier is a redeploy, not a rewrite. Guardrails: AWS Budgets alert from day one, versioned artifact prefixes for one-line rollback, read-only IAM on the serving role, and the free-account plan (hard $0) for the demo window. Prices are us-east-1 list, July 2026; LLM inference varies with usage and provider.
