System Architecture

Two planes, one contract: the offline Deep Learning pipeline produces versioned artifacts; the always-on serving plane only reads them. The whole product runs on one machine with docker compose up, and the same containers deploy to the cloud unchanged.

The Algorithms, In Plain Language

What each piece is, and how it works — no math required.

Deep learning on graphs (the core)

Money laundering and bid rigging are relationship crimes — the signal is in who connects to whom. Graph Neural Networks (GNNs) are deep-learning models that read those connections directly.

Graph Neural Network (the idea)
foundation of GraphSAGE · GATv2 · R-GCN

In simple words: every account or company starts with a profile of numbers. A GNN repeatedly lets each node look at its neighbours and blend their profiles into its own.

How it works: after two rounds of this "message passing", a node's profile summarizes its whole 2-hop neighbourhood — so a node sitting inside a suspicious ring looks different from an identical node in normal traffic. A final layer turns that profile into a risk score.

GATv2 — graph attention (headline model)
Elliptic++ transaction graph

In simple words: not all neighbours matter equally. GATv2 lets each node pay more attention to the neighbours that look most relevant, and learns those attention weights itself.

How it works: for every connection it computes an importance score, then blends neighbours weighted by importance. The attention weights double as a first hint of explanation — which edges the model cared about.

GraphSAGE — direction-aware sampling & aggregation
baseline GNN · transfer-encoder for cross-domain tests

In simple words: a simpler, fast GNN: average what flows in and what flows out separately — because money arriving and money leaving mean different things.

How it works: two separate aggregators (incoming / outgoing edges) feed a small neural layer; stacking two of these covers a 2-hop neighbourhood.

R-GCN — relational GNN
procurement graphs (firms · tenders · awards)

In simple words: procurement graphs have different kinds of links ("won this tender", "bought from"). R-GCN learns a separate blending rule per link type instead of treating them all the same.

How it works: one weight matrix per relation type (and per direction); a firm's profile mixes tender-links and buyer-links through their own learned transformations.

Graph autoencoders — DOMINANT & GAE (unsupervised arm)
no labels needed

In simple words: teach a network to compress and rebuild the graph. Whatever it rebuilds badly is unusual — an anomaly candidate, found without any labels at all.

How it works: an encoder GNN squeezes each node to a small code; a decoder tries to reconstruct the node's features and its connections from the code. The reconstruction error is the anomaly score. (Measured honestly: on our data these score below the supervised models — reported as a finding, kept for coverage.)

Focal loss — training under extreme imbalance
all supervised GNNs (~2% positives)

In simple words: with 98 innocent nodes for every flagged one, a lazy model just says "innocent" always. Focal loss turns down the reward for easy answers so training concentrates on the hard, rare cases.

How it works: the loss of every example is scaled down the more confidently the model already gets it right — easy examples fade, rare illicit ones dominate the gradient. Class-weighted cross-entropy is the simpler alternative we measure against.

From scores to an alert queue

Analysts don't review 200,000 scores — they review a short, ranked queue of groups.

Isotonic calibration + ensemble fusion
combining GNNs, autoencoders and the structural floor

In simple words: different models speak different "score languages". Calibration converts each one into honest probabilities first; only then are they averaged — so a weak model can't shout over a strong one.

How it works: isotonic regression fits a step-shaped, always-increasing curve mapping raw scores to observed probabilities on held-out data; the calibrated probabilities are then combined by weighted average. (We keep naive rank-averaging in the paper as the measured failure mode it is.)

Leiden community detection
turning node scores into case-shaped alerts

In simple words: collusion is a group activity, so alerts should be groups. Leiden finds tightly-knit clusters in the graph — the candidate "rings" and "cartels".

How it works: it repeatedly moves nodes between communities to maximize how much denser each community is than random chance, with a refinement step that guarantees well-connected results. Each community's member scores roll up into one alert risk score.

NMS dedup + budget metrics
the queue the analyst actually sees

In simple words: near-duplicate alerts waste analyst time, so overlapping alerts are suppressed; then quality is measured exactly where it matters — precision inside the top-k an analyst can actually review.

How it works: walking the queue from the top, any alert sharing too many members with an accepted one is suppressed (non-maximum suppression, borrowed from computer vision). Headline metrics are Precision@budget and AUC-PR always read against the prevalence baseline.

XGBoost baselines (the honesty yardstick)
B2 tabular · B3 tabular + graph features

In simple words: before claiming deep learning helps, we field the strongest simple competitor: gradient-boosted decision trees, with and without hand-made graph features. On Elliptic++ the trees still win — we say so, with a significance test.

How it works: hundreds of small decision trees built one after another, each correcting the previous ones' mistakes — the classic tabular workhorse, and the graph-benchmark community's required baseline.

Explaining every alert

A score without a reason is useless to an investigator — and, for AI used in fraud screening, explanations are a regulatory requirement.

PGExplainer — learned subgraph explanations
every top alert on the financial graph

In simple words: "which few edges made the model raise this alert?" PGExplainer learns to highlight the minimal set of connections that drive the prediction — remove them and the alert collapses; keep only them and it stands.

How it works: a small helper network is trained to mask graph edges while keeping the GNN's prediction intact; the surviving edges form the explaining subgraph. We verify each explanation with fidelity checks (does removing/keeping those edges really change the prediction?).

Motif matcher + FATF/OECD red flags
rule-based, deterministic, independent of the models

In simple words: known collusion has known shapes — fan-in "smurfing", circular round-tripping, bid rotation, cover bidding. A pattern matcher searches each alert for these nine shapes and cites the official FATF / OECD red-flag indicator each one corresponds to.

How it works: hand-written graph rules (cycles, stars, chains, cliques, rotation statistics) run over the alert's subgraph; matches are validated against injected synthetic patterns (100% recall on all nine families) so the matcher is trusted independently of the neural models.

The guarded LLM copilot

Generative AI joins only at the investigation surface — it never finds or ranks anything.

Bounded SQL-agent loop
NVIDIA NIM LLM over the artifact store

In simple words: the copilot answers questions like "why is alert #37 ranked above #12?" by looking things up, not by remembering or guessing — it writes read-only SQL against the served queue and quotes what it finds.

How it works: the LLM proposes a tool call (SQL query, alert lookup, red-flag search); the system executes it and feeds the result back; after a bounded number of steps the answer is composed strictly from gathered evidence, with the trace shown in the UI.

Deterministic guardrails + goldens gate
every response, every time

In simple words: two hard filters run on every reply: any number must literally appear in the tool evidence (no invented figures), and guilt-asserting phrasing is rewritten into screening language. A 24-question "goldens" exam gates releases — currently 24/24 with zero guilt-language violations.

How it works: a regex-level numeric gate cross-checks the answer against evidence text; a rewrite table converts phrases like "is guilty of" into "shows flagged patterns consistent with"; BM25 keyword retrieval forces typology questions to cite the curated FATF/OECD corpus verbatim.

Statistical rigor (why the numbers can be trusted)

The evaluation protocol is itself a contribution: every headline number ships with its uncertainty.

Multi-seed protocol + paired bootstrap
all headline experiments (≥5 seeds, 2,000 resamples)

In simple words: train the same model five times with different random starts and report the spread — one lucky run is not a result. To compare two models, resample the same test nodes thousands of times and check the gap survives.

How it works: stratified bootstrap resampling keeps the class balance; identical resamples feed both models so the difference distribution and its confidence interval are honest; a p-value falls out directly.

Transfer matrices + robustness curves
LOCO / LOMO across countries & markets · label-noise · label-efficiency

In simple words: "does what we learned in six countries work in the seventh we never trained on?" — every country and market takes one turn being held out. Robustness curves then stress the pipeline: corrupt training labels on purpose, shrink the label budget, sweep the protocol knobs — and report what breaks and what doesn't.

How it works: entity-disjoint folds guarantee the held-out market shares nothing with training; results are read as lift over prevalence so different markets stay comparable; each curve point is a full retrain at a controlled corruption or budget level.
