How Data Flows

From public datasets to a decision — eight simple steps.

1

Download the datasets

Six public datasets — Bitcoin transactions, a simulated banking world, and European procurement records — are downloaded and checksum-verified. Raw data stays on disk, never in the code repository.

2

Turn everything into one graph

Each dataset becomes the same simple picture: dots (accounts, companies) connected by lines (payments, contracts). One format means one model family works for both crime domains.

3

Split by time — no cheating

The models train only on the past and are tested only on the future, exactly like real life. Automated tests make it impossible for tomorrow's information to leak into today's training.

4

Deep learning scores every node

Graph Neural Networks read each account's connections and give it a risk score. Classic methods (decision-tree models) run alongside as a fairness check — and we report honestly when they win.

5

Group suspects into alerts

Collusion is a team crime, so suspicious nodes that are tightly connected get grouped into one alert. Alerts are ranked, near-duplicates removed, and only a short reviewable list remains.

6

Attach evidence to every alert

For each top alert the system highlights the few connections that triggered it, names the known pattern it matches (e.g. “fan-in”, “bid rotation”), and cites the official FATF / OECD red flag it corresponds to.

7

Publish results as simple files

The pipeline writes its results — alerts, evidence, accuracy reports — as plain files and exits. The dashboard and API only ever read these files; they can never change a score.

8

A human decides

The investigator reviews the ranked list, opens the evidence, asks the Copilot questions — and makes the escalation decision. The system only screens; it never judges.

The one rule everything obeys: the deep-learning side writes files, the product side reads files — nothing else crosses between them. That's why serving is safe, cheap, and can never “retrain itself”.

The Algorithms, Explained Simply

What each piece does and how it works — in everyday language.

Graph Neural Networks — the heart of the system used everywhere risk is scored

What it does: gives every account or company a risk score by looking at who it deals with, not just what it looks like.

How it works: each node starts with a simple profile of numbers. In a few rounds, every node mixes its neighbours' profiles into its own — so after two rounds, a node “knows” about its whole neighbourhood. A node sitting inside a laundering ring ends up looking measurably different from a normal one.

GATv2 — attention: not all neighbours matter equally the main model on the Bitcoin graph

What it does: learns which of a node's connections are the important ones and listens to those the most.

How it works: for every connection it computes an importance weight, then blends neighbours accordingly. Those weights double as a first clue for explanations — they show which links the model cared about.

GraphSAGE & R-GCN — the direction and type of a link matter GraphSAGE on financial data · R-GCN on procurement data

What it does: treats “money coming in” differently from “money going out” (GraphSAGE), and “won a tender” differently from “bought from” (R-GCN).

How it works: each direction — and each type of relationship — gets its own learned mixing rule instead of throwing everything into one pot.

Focal loss — learning when 98% of examples are innocent training every deep model

What it does: stops the model from taking the lazy shortcut of calling everything “innocent” (which would be 98% correct and 100% useless).

How it works: during training, easy examples the model already gets right earn almost no credit, so all the learning effort concentrates on the rare, hard, suspicious cases.

Autoencoders (DOMINANT & GAE) — finding odd nodes without any labels the unsupervised safety net

What it does: flags unusual nodes even where no one has ever labelled a crime.

How it works: a network learns to compress and then rebuild the graph. Whatever it rebuilds badly is unusual — an anomaly candidate. (Honest result: on our data these score below the supervised models, and we say so.)

Calibration + ensemble — combining models fairly turns many model outputs into one risk score

What it does: merges several models' opinions without letting a loud weak model drown out a quiet strong one.

How it works: each model's scores are first converted into honest probabilities (checked against held-out data), and only then averaged. We also show what happens if you skip this step — the ensemble collapses.

Leiden grouping + de-duplication — from scores to a short queue builds the alert list the investigator sees

What it does: turns two hundred thousand scores into a few hundred group-shaped alerts, because collusion is done by groups.

How it works: a community-finding algorithm spots tightly-knit clusters; each cluster's scores roll up into one alert; overlapping alerts are removed top-down so the queue never wastes the investigator's time twice.

XGBoost baselines — the honesty yardstick every headline comparison

What it does: before claiming deep learning helps, we field the strongest classic method (boosted decision trees) on the same data.

How it works: hundreds of small decision trees, each correcting the last. On the Bitcoin data the trees still win — we report that openly, with a statistical significance test, instead of hiding it.

PGExplainer — “show me why” evidence for every top alert

What it does: answers “which few connections made the model raise this alert?”

How it works: a small helper network learns to keep only the connections that the prediction truly depends on. We then double-check every explanation: remove those links and the alert should collapse; keep only them and it should stand.

Pattern matcher + official red flags — evidence a regulator recognises runs on every alert, independent of the AI

What it does: checks each alert against nine known collusion shapes — like many small deposits converging on one account, or companies taking turns to win tenders — and cites the official FATF / OECD red-flag each shape corresponds to.

How it works: hand-written rules (no AI involved) search the alert's neighbourhood for these shapes, so this evidence stands even if you distrust the neural network entirely.

The Copilot's guardrails — an AI that can't make things up wrapped around every Copilot answer

What it does: lets investigators question the system in plain English without the usual AI risks.

How it works: the Copilot may only read the published results; every number in its answer must literally appear in what it looked up (otherwise the answer is blocked); and phrases like “is guilty” are automatically rewritten into “shows patterns consistent with…”. A 24-question exam gates every release — currently a perfect score.

Statistical rigor — why the numbers can be trusted behind every published result

What it does: makes sure no result is a lucky accident.

How it works: every headline model is trained five times with different random starts and reported with its spread; model comparisons are re-checked thousands of times on resampled data; and each country or market takes a turn being fully held out to prove the system works on places it has never seen.
